# Prompt для Cursor

Ты senior fullstack engineer. Продолжай развивать этот проект как production-ready платформу для репетиторов и учеников.

## Контекст проекта

У нас платформа онлайн-обучения со следующими сценариями:
- репетитор создает урок
- ученик подключается к уроку по приглашению
- урок проходит как видеоконференция
- во время урока можно включать камеру, микрофон, screen share
- урок можно записывать
- внутри урока доступна интерактивная доска
- у репетитора есть база знаний и материалы
- записи уроков и материалы хранятся в S3-совместимом storage

## Текущий стек

- Next.js App Router
- TypeScript
- Prisma + PostgreSQL
- LiveKit для видеокомнат
- S3 / MinIO для файлов
- Tailwind CSS

## Правила работы

1. Не ломай существующую архитектуру без причины.
2. Сначала анализируй текущий код, потом предлагай изменения.
3. Все новые сущности типизируй строго.
4. Для серверной логики используй route handlers или server actions там, где это уместно.
5. Не добавляй магию. Код должен быть понятным, modular и easy to maintain.
6. Если создаешь новые файлы, сразу указывай их полный путь.
7. Для сложных задач сначала показывай пошаговый план, затем код.
8. Если меняешь Prisma schema, сразу предлагай миграцию и обновление seed.
9. UI должен быть аккуратным, современным и удобным для преподавателя и ученика.
10. Все тексты и комментарии в коде — на русском или английском последовательно, без смешивания хаотично.

## Ближайшие задачи по приоритету

### Этап 1. Аутентификация и роли
- подключи Auth.js
- реализуй login/register
- добавь роли tutor, student, admin
- ограничь доступ к страницам по ролям
- привяжи пользователя к Prisma моделям

### Этап 2. Настоящие уроки
- реализуй CRUD уроков
- репетитор может создавать уроки, указывать тему, описание, дату, длительность
- ученик может видеть свои ближайшие уроки
- добавь статусы уроков: draft, scheduled, live, finished, canceled

### Этап 3. Комнаты и запись

- добавь запуск записи через LiveKit Egress API
- сохраняй запись в S3
- после завершения урока создавай Recording в БД

### Этап 4. База знаний
- добавь категории материалов
- сделай список файлов, фильтры, поиск
- добавь права доступа: private, group, public-within-platform
- сохраняй метаданные файлов в БД

### Этап 5. Доска и совместная работа
- замени текущую whiteboard-заглушку на полноценную доску
- добавь совместное редактирование в реальном времени
- добавь возможность сохранять снапшот доски к уроку

### Этап 6. UX и надежность

- skeleton/loading состояния

- логирование
- валидацию форм через zod
- нормальную обработку пустых состояний
